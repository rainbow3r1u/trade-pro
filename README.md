# 加密货币策略扫描器 (Crypto Scanner V2)

本项目是一个本地与云端可用的加密货币策略扫描器，包含前端界面和后端服务。目前支持的基础功能和核心策略如下：

## 核心策略
- **稳步抬升 (Strategy 1)**：通过多小时的阳线连涨过滤震荡上行的币种。
- **稳步抬升 PRO**：在原有策略基础上增强了过滤条件（实体比例 > 40%，且剔除单根涨幅超过 8% 的“加速赶顶”行情），带有资金异动观察窗（Watchlist）。
- **圆弧底突破 (Arc Bottom)**：支持模糊参数匹配的“茶杯柄/圆弧底”形态策略。
- **其他内置工具**：如成交量、均线聚拢、6小时数据记录、以及 10% 暴涨监控。

## 启动服务器
本项目运行在 `5002` 端口，采用后台常驻运行方式。
如果服务器重启或进程意外终止，可以通过以下命令重新启动网站：

```bash
nohup python3 app.py > web_server.log 2>&1 &
```

*(该命令会将程序挂载到后台，并将所有日志输出到 `web_server.log` 中。)*

## 常用 Git 知识与操作指南

### 1. 什么是 `origin`？
在 Git 中，**`origin` 是一个系统默认的“远程仓库代号/别名”**。
当你把本地代码推送到 GitHub 时，输入完整的网址会非常麻烦。因此，Git 会用一个简短的名字来代替长长的网址。
在本项目中，`origin` 就等价于您的远程仓库地址：
`git@github.com:rainbow3r1u/crypto-scanner-v2.git`

当你执行 `git push origin main` 时，意思就是：
> “把我的本地 `main` 分支代码，推送到名叫 `origin` 的 GitHub 仓库上去。”

### 2. 日常提交流程
当你在服务器上修改了代码（例如新增了功能或修复了 BUG），想保存并同步到 GitHub 时，按顺序执行以下三步即可：

**第一步：将改动添加到暂存区**
```bash
git add .
```
*(代表将当前目录下的所有变化加入提交队列。)*

**第二步：提交改动并写上备注**
```bash
git commit -m "feat: 这里写上你改了什么，例如添加了README"
```

**第三步：推送到 GitHub (origin)**
```bash
git push origin main
```
*(此时代码就会同步到云端的 V2 仓库中了！)*
