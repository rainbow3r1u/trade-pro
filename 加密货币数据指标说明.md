# 加密货币核心数据指标

## 流通量 (Circulating Supply)

**定义：** 市场上真正可以自由交易的币的数量。不是总发行量 (Total/Max Supply)。

**为什么重要：**
- 计算市值的分母：市值 = 价格 × 流通量
- 锁仓（团队、基金会、质押）的币不计入流通量
- 流通量变化（解锁/销毁）直接影响价格

**影响因素：**
- 解锁事件（私募/团队解锁 → 流通量增加）
- 销毁机制（通缩模型 → 流通量减少）
- 质押/解质押周期

---

## 市占率 (Dominance)

**定义：** 单个币的市值占全网总市值的百分比。

```
市占率 = 币市值 ÷ 全网总市值 × 100%
```

**关键阈值与市场阶段：**

| BTC市占率 | 市场状态 | 策略建议 |
|-----------|---------|---------|
| > 60% | BTC主导，山寨低迷 | 主做BTC相关，少碰山寨 |
| 50-60% | 中性区间，震荡 | 精选标的 |
| 40-50% | 山寨季早期 | 关注轮动，BB_CLIMB信号增多 |
| < 40% | 山寨季高潮 | 短线脉冲机会多，注意风险 |

**常见误区：**
- BTC 涨 = BTC市占率不一定涨（如果ETH涨更多，BTC市占率反而会跌）
- 市占率下降 ≠ 熊市，可能是山寨牛

---

## 换手率 (Turnover)

**定义：** 24小时交易量与市值的比率。衡量资产流动性。

```
换手率 = 24h交易量 ÷ 市值
```

| 换手率 | 含义 | 交易策略 |
|--------|------|---------|
| < 0.05 (5%) | 极低流动性 | 大单易滑价，不适合短线 |
| 0.05-0.2 (5-20%) | 正常 | 多数主流币区间 |
| 0.2-0.5 (20-50%) | 活跃 | 短线交易良好标的 |
| 0.5-1.0 (50-100%) | 极活跃 | 可能有大资金进场，VOL_SURGE候选 |
| > 1.0 (>100%) | 异常高 | 注意是否为出货/对倒 |

**换手率变化的意义：**
- 换手率从低到高骤增 → 有资金关注，可能启动
- 换手率持续高位 → 交投活跃，趋势延续概率大
- 换手率从高到低回落 → 热度退潮，可能见顶
- 价格涨 + 换手率降 → 缩量上涨，可能虚涨

---

## 三者的关系

### 公式链

```
市值 = 流通量 × 当前价格
换手率 = 24h交易量 ÷ 市值
市占率 = 币市值 ÷ 全网总市值
```

### 互相影响的场景

**场景 1：解锁抛售**
- 流通量增加（解锁）→ 市值不变的情况下价格下跌
- 如果伴随换手率骤增 → 可能是筹码派发，短线要小心

**场景 2：庄家吸筹**
- 换手率小幅上升 + 价格横盘 → 有人默默买入
- 市占率不变或微升 → 资金净流入

**场景 3：VOL_SURGE 的有效性判断**
- 成交量突增 + 换手率正常偏高 (0.2-0.5) → ✅ 正常脉冲信号
- 成交量突增 + 换手率极低 (< 0.05) → ⚠️ 可能是小市值币对倒作量
- 成交量突增 + 换手率 > 1.0 → ⚠️ 过热，追高风险大

### 在你的策略中的应用

**BB_CLIMB（定方向）：**
- 市占率判断资金流向 → 决定做BTC还是做山寨
- 流通量变化辅助判断趋势持续性（大解锁前谨慎做多）

**VOL_SURGE（抓脉冲）：**
- 换手率作为辅助筛选条件，过滤低流动性标的
- 换手率 + 成交量双突增 → 更可靠的信号

## 数据源

| 数据项 | 数据源 | 获取方式 |
|--------|--------|---------|
| 价格、交易量 | 币安 API | 已有 WebSocket 实时 |
| 换手率 | CoinMarketCap API | `turnover` 字段 |
| 流通量 | CoinMarketCap API | `circulatingSupply` 字段 |
| 市占率 | CoinMarketCap API | `dominance` 字段 |
| 24h涨跌幅 | 币安/CMC | 两者都有 |

### CoinMarketCap 免费接口

```python
import requests

# 全币种排名（按市值排序）
url = "https://api.coinmarketcap.com/data-api/v3/cryptocurrency/listing"
params = {
    "start": 1,
    "limit": 100,
    "sortBy": "market_cap",
    "sortType": "desc",
    "convert": "USD"
}
resp = requests.get(url, params=params)
data = resp.json()["data"]["cryptoCurrencyList"]

# 每个币包含:
# price, volume24h, marketCap, circulatingSupply
# percentChange1h/24h/7d/30d/90d/1y
# dominance, turnover, fullyDilluttedMarketCap
```

**注意：** CMC 的免费接口不需要 API Key，但请求不要太频繁（建议每 5-10 分钟拉一次）。
